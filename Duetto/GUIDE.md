# Duetto 功能与操作指南

*listen together, in tune.*

Duetto 是一个可以自己部署的「一起听歌」播放器：两个人（或一个人和 TA 的 AI）共享一个房间，
同一首歌、同一秒进度。内置的 AI 伙伴不是聊天挂件——它**真的听过**你们放的每一首歌，
记得你们为哪句歌词停留过，并且为每首歌长出属于你们的回忆。

---

## 一、部署

> 需要 **Node 24 或更高**（用到了 Node 内置的 SQLite）。

```bash
git clone https://github.com/avisforevelyn/Duetto.git
cd Duetto
npm install
npm start
```

浏览器打开 `http://localhost:4183/pkg/index.html`。换端口：`PORT=8080 npm start`。

放到公网（可选）：反向代理指向 4183，记得代理 `/ws`（房间实时同步用）。
建议用 HTTPS——iOS 的后台播放、PWA 安装都需要它。

**安全提醒**：Duetto 目前是单用户 / 可信房间架构，不自带应用登录。`?room=暗号` 只是在
同一服务器里分房间，不是访问控制。公网部署前请先加一层自己的门禁，例如反向代理 Basic Auth、
Cloudflare Access、Tailscale 或只放在私有网络里；否则外部访客可以调用同一套 API，读房间/档案，
也可能触发网易云账号操作。

**添加到主屏幕（推荐）**：手机浏览器里「添加到主屏幕」，Duetto 会以独立 PWA 运行，
锁屏可控制播放，媒体卡片显示封面歌名。

## 二、首次配置

0. **设门禁 PIN**：第一次打开 Duetto 会让你设一个 PIN（至少 4 位）——这是**应用自己的锁**：
   设了之后所有接口和房间连接都需要它换来的通行证，别人拿到网址也进不来。
   新设备打开时输一次 PIN 即可。忘了的话删掉服务器上的 `data/auth.json` 重设。
   （注意：下面的网易云扫码登的是你的网易账号，管的是歌单和音源，和这道门是两回事。）
1. **网易云登录**：曲库页 → 扫码登录（用网易云 App 扫）。登录后有你的歌单、日推、私人 FM、红心同步。
2. **AI 配置**：一起听 tab → 模型设置：
   - **聊天模型**：填 OpenAI 兼容端点 + API Key，点「拉取模型列表」选一个；
   - **分析模型**（可选）：AI「听歌」用的模型（需支持音频输入，如 Gemini 系）。
     随便填多少——只填模型名、只填 Key 都行，缺的自动共用聊天配置；全空则用聊天模型；
   - **AI 人设**：TA 是谁，你说了算；
   - **对话风格**：TA 怎么说话（例：「回复简短像聊微信、多用语气词」）。留空用默认；
   - **昵称**：你和 TA 的名字，全应用的称呼、状态卡片都跟着它。
3. 部署者也可以直接在服务器 `data/settings.json` 里配置 `ai.base_url / api_key / model /
   a_model / style / context_url`——配好后另一个人打开页面即用，前端什么都不用填。

## 三、播放器

- **此刻页**：封面/歌词双页横滑切换；点歌手名进歌手主页（热门 50 首，可整批播放或加入列表）；
  点歌名进歌曲详情（在场记录 / 歌曲分析 / 网易评论）。
- **歌词页**：
  - **点一句** → 问 Ta（引用这句发起对话）；
  - **长按一句** → 引用它，去房间发消息时自动带上（微信式引用块）；
  - 上下滑动选句，松手贴线，点小播放键从那句开始；
  - 右上角开关译文；四种歌词字体、字号可调。
- **播放模式**：列表循环 / 单曲循环 / 随机。接下来**三首**的播放地址在开播时就预取好
  （随机模式会提前抽好三步的随机顺序），锁屏、后台连续切歌全程不需要网络。
- **私人 FM**：电台快放完会自动请求新歌接上，同样提前备好地址。
- **本地歌**：曲库 → 本地，粘贴音频直链即可播放。
- **歌单批量操作**：歌单详情页多选后可批量播放 / 加入待播 / 收藏到歌单 / 删除。

## 四、一起听房间

**进入**：歌词页再往左滑一次。所有打开同一服务器的人默认在 `main` 房；
URL 加 `?room=你们的暗号` 开私密房间。

- **实时同步**：播放、暂停、进度、切歌，房间里所有人同步。
- **聊天**：底部输入框边听边说。AI 的每次回复都带着完整上下文（见第五节）。
- **引用歌词**：悬浮球展开 → 歌词 → **长按一句**，输入框上方出现引用条（× 可取消），
  发送后气泡里是微信式的引用块，这句也会记进这首歌的档案。
- **状态卡片**：谁播放了、暂停了、红心了、收藏了什么，居中小卡自动记录
  （锁屏、来电这类系统行为不算，不会刷屏）。在播放器页操作同样会记录。
- **分享歌**：顶栏分享键（或让 AI 分享）——歌以卡片形式进聊天，点卡即播；
  AI 分享的歌会自动加入播放列表并开始播放。
- **悬浮球**：可拖动；展开后是迷你播放器 + 歌词 / 点歌 / 歌单 / 待播四个页签。
- **房间设置**（右上角 ⋯）：
  - **回复样式**：分气泡（微信式短消息）/ 完整带思考（整段回复 + 可展开的「💭 思考过程」）；
  - **顶栏样式**：完整卡（双头像耳机线 + 白色播放卡）/ 极简条；
  - 房间背景、隐藏状态卡、隐藏气泡头像、时间感知开关；
  - 气泡颜色 / 透明度 / 磨砂（你和 TA 各一套）；
  - 顶栏、顶卡、状态卡、输入框、分享卡、背景蒙版——六个表面各自的透明度与磨砂。

## 五、AI 是怎么「听歌」的

- **真听**：你们在房间里放一首没听过的歌并开口聊它（或点了问 Ta），Duetto 会把
  **你正在听的那条音频流**下载下来喂给分析模型真的听一遍，写下分时间段的听后赏析
  （前奏在哪、情绪怎么走、哪句最戳）。一首歌只听一次，永久缓存。
  端点不支持音频输入时自动退回完整时间轴歌词分析。
  首次聊一首新歌时回复会等分析完成（约半分钟到一分半），之后秒回。
- **每次对话 AI 都知道**：正在放哪首、放到第几秒、**当前唱到哪一句**、你们一起听过几次、
  它的听后赏析、这首歌的回忆，以及房间最近 12 条对话。
- **在场记录 → 印象**：你在房间聊某首歌的每一轮、问 Ta 的每次问答，都挂在这首歌上；
  **每满 6 条**自动揉成一段 150 字内的第一人称回忆，之后在旧回忆上续写、不推翻。
  歌曲详情页的「在场记录」和「歌曲分析」标签页都能看到。
- **DJ 能力**：聊天中让 TA 放歌、切歌、暂停、继续、分享、红心、加待播队列——直接说就行。
- **外部记忆接口**：`data/settings.json` 配 `ai.context_url` 指向你自己的记忆/RAG 服务，
  每轮对话 POST `{message, song, user, ai}`，返回的 `{context}` 文本会注入提示词。
- **提示词透明**：浏览器打开 `/api/prompt-preview`（`?mode=stream` 看完整带思考版）
  能看到发给模型的完整 system 提示词。

## 六、问 Ta 与听歌档案

- **问 Ta**：歌曲详情页的「问 Ta」按钮，或歌词页点任意一句。可以用四个快捷问题
  （这句在说什么 / 我想到你 / 记住这一刻 / 陪我听完），也可以写自己的话。
  问答会出现在房间时间线里，并成为这首歌的在场记录。
- **听歌档案**（一起听 tab）：一起听过多少次、多少首歌、最常在什么时段听；
  下面是全部在场记录，带封面，点卡片进歌曲详情。

## 七、皮肤与美化

- **五款皮肤**：凝脂（暖白）、豆青（浅绿）、雪青（蓝紫）、藕荷（粉调）、霁蓝（雾蓝），
  外加完全自定义调色盘（主色 + 各表面颜色单独调）。
- 壁纸、头像：一起听 tab 上传，存在浏览器本地（IndexedDB），不上传服务器。
- 界面各表面（底栏、卡片、搜索框……）透明度与磨砂在装扮设置里逐项可调。

## 八、更新到新版本

```bash
git pull
npm install
# 重启服务（systemctl / pm2 / 你自己的方式）
```

更新不会动你的数据——设置、PIN、听歌档案都在 `data/` 目录里，这个目录被 .gitignore 忽略、
`git pull` 根本不碰它。数据库表用「不存在才创建」，旧库拉新代码后照常衔接。前端带版本号缓存，
刷新页面即是新版。（若你自己改过代码，pull 可能提示冲突，那是你的改动，按需处理即可。）

## 八、数据说明

长期档案存在服务器 `data/` 的 SQLite（`listen.db`）：听歌流水、歌（含歌词缓存与回忆）、
在场记录、听后分析、房间消息。JSON 只存瞬时状态（配置、登录 cookie、封面缓存）。
`data/` 整个目录被 .gitignore 忽略——你的一切都只在你自己的服务器上。

## 九、常见问题

- **有些歌播不了 / 分析听错歌**：无版权曲目（如部分周杰伦）依赖你登录的网易云账号权益；
  分析永远听「你正在播放的那条流」，你听到什么它就分析什么。
- **iOS 后台不切歌**：确认用「添加到主屏幕」的 PWA 打开，且服务在 HTTPS 下。
- **多设备同房刷卡片**：只有操作者的设备会发状态卡，其他设备只显示——放心多开。
- **回复带思考链但看不到**：房间设置 → 回复样式 → 完整带思考；气泡上方的「💭 思考过程」点开。
